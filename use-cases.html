📋 Scenario: Debugging Pandas Errors in JupyterLab

The Problem: You're analyzing data in JupyterLab. A pandas operation throws a cryptic error about data types or indices.

  1. Open JupyterLab in your browser and load your dataset
  2. Run data transformation: df.merge(df2, on='user_id')
  3. Cell fails with KeyError or ValueError
  4. Select the error traceback in the cell output
  5. Click AI Code Buddy hover button
  6. AI diagnoses the issue and provides corrected pandas code
  7. Apply fix and continue your analysis
# In JupyterLab cell: result = df.merge(df2, on='user_id', how='inner') KeyError: 'user_id' During handling of the above exception, another exception occurred: ... [Select error → AI Code Buddy analyzes] # AI discovers the issue and suggests: # Check actual column names first: print("df columns:", df.columns.tolist()) print("df2 columns:", df2.columns.tolist()) # AI explains: "Column is likely 'userId' (camelCase) not 'user_id'" result = df.merge(df2, left_on='userId', right_on='userId', how='inner') # Or rename for consistency: df2 = df2.rename(columns={'userId': 'user_id'}) result = df.merge(df2, on='user_id', how='inner')

📋 Scenario: Complex SQL Query Optimization

The Problem: You're using phpMyAdmin to analyze slow queries. The EXPLAIN output shows performance issues, but you're not sure how to optimize it.

  1. Open phpMyAdmin in your browser and run your query
  2. Click "Explain" to see the query execution plan
  3. Select the EXPLAIN output showing "Using filesort" or "Using temporary"
  4. Click AI Code Buddy hover button
  5. AI analyzes and suggests: missing indexes, query rewrites, schema improvements
  6. Apply AI suggestions and re-test query performance
-- Original slow query SELECT u.*, COUNT(o.id) as order_count FROM users u LEFT JOIN orders o ON u.id = o.user_id WHERE u.created_at > '2024-01-01' GROUP BY u.id ORDER BY order_count DESC; [Select EXPLAIN showing "Using filesort; Using temporary"] # AI Code Buddy suggests: -- Add composite index for better performance: CREATE INDEX idx_users_created ON users(created_at, id); CREATE INDEX idx_orders_user ON orders(user_id); -- Optimized query with materialized CTE: WITH user_orders AS ( SELECT user_id, COUNT(*) as cnt FROM orders GROUP BY user_id ) SELECT u.*, COALESCE(uo.cnt, 0) as order_count FROM users u LEFT JOIN user_orders uo ON u.id = uo.user_id WHERE u.created_at > '2024-01-01' ORDER BY order_count DESC;
