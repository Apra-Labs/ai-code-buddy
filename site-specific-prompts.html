What Are Site-Specific Prompts?

Site-Specific Prompts let you configure different AI behavior for different websites. The AI automatically adapts its expertise based on where you're working‚Äîwithout you having to switch settings manually.

Think of it as giving your AI assistant different "hats" to wear. A DevOps expert for build logs, a Linux guru for remote administration, a code reviewer for GitHub, and a log analyst for monitoring dashboards.

How to Set Up Site-Specific Prompts

Follow these steps to configure context-aware AI behavior for different websites:

1. Open Extension Settings
Click the AI Code Buddy extension icon and navigate to the Advanced tab
‚Üì
2. Find Site-Specific Prompts Section
Scroll down to the "Site-Specific System Prompts" section
‚Üì
3. Click "Add Site-Specific Prompt"
A modal will open with three fields to configure
‚Üì
4. Enter Pattern
Specify which websites this prompt applies to using exact domain or wildcards (e.g., *github.com or *.azure.com)
‚Üì
5. Add Name (Optional)
Give it a friendly name like "DevOps Build Logs" or "Linux Administration"
‚Üì
6. Write Your System Prompt
Describe the AI's role, expertise, and how it should respond. Be specific about what you want it to focus on
‚Üì
7. Save and Test
Click Save, visit a matching website, and use the AI. It will automatically apply your custom prompt!

üéØ Pattern Matching Examples

Supported Pattern Types

  • Exact Domain: github.com Matches only github.com (not subdomains)
  • Wildcard Subdomain: *.github.com Matches gist.github.com, api.github.com, etc. (not github.com itself)
  • Wildcard Prefix: *github.com Matches github.com AND all subdomains
  • Azure Blobs: *blob.core.windows.net Matches any Azure storage account
  • AWS Consoles: *console.aws.amazon.com Matches all AWS console pages
üí° Tip: When multiple patterns match, the most specific one wins. An exact domain match beats a wildcard pattern.

üí° Pro Tips

  • Be Specific: The more detailed your prompt, the better the AI performs. Include output format preferences, terminology, and focus areas.
  • Use Examples: Include examples in your prompts to show the AI exactly what kind of responses you want.
  • Test and Iterate: Start with a basic prompt and refine it based on the AI's responses. You can edit prompts anytime.
  • Enable/Disable: Use the toggle switches to temporarily disable prompts without deleting them.
  • Export Your Prompts: Use the Export Configuration feature in Advanced tab to backup and share your prompts.
  • Default Fallback: Set a "Default System Prompt" that applies when no site-specific pattern matches.
