Managing Site Permissions

Control where AI Code Buddy runs

Overview

By default, AI Code Buddy runs on all websites to detect text inputs and command outputs wherever you need them. However, Chrome's built-in extension permissions system lets you control exactly which sites can use the extension.

This guide shows you how to:

  • Allow only specific sites (e.g., only your RPort servers)
  • Block specific sites (e.g., social media, Gmail)
  • Temporarily disable the extension on certain pages

Option 1: Allow Only Specific Sites

Perfect if you only want AI Code Buddy on your RPort server(s) or specific work tools.

1 Open Chrome Extensions
  • Navigate to chrome://extensions/ in Chrome
  • Or click ⋮ (three dots) → Extensions → Manage Extensions
2 Find AI Code Buddy

Locate "AI Code Buddy by Apra Labs" in your extensions list

3 Click "Details"

Click the "Details" button under the extension

4 Change Site Access

Scroll down to "Site access" section

Click on the dropdown and select: On specific sites

5 Add Your Sites

Click "Add a new page" and enter your site URLs:

Example: Allow Only RPort Servers

If your RPort servers are at:

  • https://rport.company.com
  • https://rport-prod.company.com

Add these exact URLs to the allowed sites list.

Tip: You can use wildcards like https://*.company.com/* to allow all subdomains.

Option 2: Block Specific Sites

If you want the extension on most sites but want to exclude a few (like Gmail, social media, etc.):

1 Visit the Site You Want to Block

Go to the website where you don't want AI Code Buddy to run (e.g., Gmail, Facebook)

2 Click the Extension Icon

Click the AI Code Buddy icon in your Chrome toolbar (top-right)

3 Right-Click for Options

Right-click on the extension icon and select:

This can't read or change site data

Or choose:

  • "When you click the extension" - Only works when you manually activate it
  • "On this site" - Always works on this specific site
  • "On all sites" - Works everywhere (default)
Note: This setting is per-site. You need to do this for each site you want to block.

Option 3: Temporarily Disable

Need to disable the extension temporarily without uninstalling?

1 Open Extensions

Go to chrome://extensions/

2 Toggle Off

Find AI Code Buddy and toggle the switch to OFF

The extension will be completely disabled on all sites

Re-enable: Just toggle the switch back to ON when you need it again.

Common Scenarios

Scenario 1: Work Only (RPort + Azure DevOps)

Goal: Only run on work-related sites

Solution: Use "On specific sites" and add:

  • https://rport.yourcompany.com/*
  • https://dev.azure.com/yourorg/*
  • https://*.visualstudio.com/*

Scenario 2: Development Tools Only

Goal: Use on GitHub, GitLab, and local dev servers

Solution: Use "On specific sites" and add:

  • https://github.com/*
  • https://gitlab.com/*
  • http://localhost/*
  • http://127.0.0.1/*

Scenario 3: Everywhere Except Social Media

Goal: Run everywhere except Facebook, Twitter, Instagram

Solution: Keep "On all sites", then block specific sites:

  1. Visit facebook.com
  2. Right-click extension icon → "This can't read or change site data"
  3. Repeat for Twitter, Instagram, etc.

Troubleshooting

Extension Not Working on a Site?

  1. Check chrome://extensions/
  2. Click "Details" on AI Code Buddy
  3. Verify "Site access" includes the site
  4. Add the site if it's missing
  5. Refresh the page

Buttons Still Appearing on Blocked Sites?

  1. Refresh the page (Ctrl+R or Cmd+R)
  2. Clear browser cache if needed
  3. Verify site access settings are correct

Can't Find "Site Access" Option?

Important: The "Site access" setting only appears if you installed the extension as a Chrome Web Store extension. If you loaded it as an "unpacked extension" (developer mode), you won't see this option.
